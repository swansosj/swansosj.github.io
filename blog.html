My Path to NetDevOps
June 8, 2022

When I was wrapping up my Bachelors at Florida State College at Jacksonville my confidence in my own learning capability had taken a boost. Having attempted a Java Course in the past my initial attitude toward learning a programming language was still that it just wasn't for me. When I took my first contract as a Network Administrator it consisted of a lot of repetitive work. Concurrently I was tinkering with penetration testing hardware such as WiFi Pineapple, LAN Turtle, and USB Rubber Ducky. The Rubber Ducky language I found to be extremely easy to learn and it dawned on me. If I look at the device as a tool to enter the mass repetitive commands I was entering I could free up a lot of my time allowing me to learn more advanced topics.

Realizing this benefit I started to ask myself if there was a way to add some logic to the automation. Everything I began reading started to point me towards Python. At the time this technology was still being developed. I mean I was watching videos in Python 2.7, Telnet libraries, and Paramiko. My question had been answered though. I could learn some basic Python and interact with a network device over SSH. Thus began my journey to start learning the language.

After learning the basic elements that make up the language I started with scripting in my lab using telnet for simplicity's and moving to Paramiko. With the release of Netmiko interacting with devices via SSH became more simplified and I began to perform mass maintenance with Python. The trend started to rise as more and more training became available, the development of network device APIs, and the growth of Cloud technologies. Cisco DevNet was a huge game changer introducing me to APIs which after a short amount of time showed their value. I purchased a few books full of fun Python projects and began populating spreadsheets, making mass updates, and integrating various platforms. When I got introduced to Flask and built my first Web Application with Python I knew this was the way of the future.

It is important to note that the traditional network engineer will still have a place in the network space. This will never go away but how they do their job and the ways in which the network is integrated into the rest of the environment will. The purpose of this post is to not only give someone interested in learning NetDevOps my own personal experience in acquiring the skills but also to give a big piece of advice to those still fighting the transition... Don't! You are not only fighting a loosing battle but revealing very selfish attributes about your character. Your willing to fight progress ultimately crippling a companies ability to evolve in the technology space so you don't have to learn something that is unfamiliar to you. If you are the individual that knows it's coming and has just been putting it off my advice to you is start now. Learning a programming/scripting language is not a sprint it is very much a journey. I look forward to collaborating with the NetDevOps Engineers of the future. Happy Coding Everyone and remember... import this
